排序一（冒泡、简单选择、直接插入、希尔）

一、基本定义

排序也叫排序算法，是将一组数据，依照指定顺序进行排列的过程

二、基本类型
  1. 内部排序：指将需要处理的所有数据都加载到内部存储器中进行排序
  2. 外部排序：数据量过大时，无法全部加载到内存中，需要借助外部存储（文件）进行排序

冒泡排序

冒泡排序属于内部排序，对待排序序列从前向后（从下标较小的元素开始），依次比较相邻元素的值，若发现逆序则交换，使数值较大的元素逐渐从前向后移动，直到没有逆序的记录为止。

冒泡排序的时间复杂度是O(n2),如果不进行优化，可能会浪费计算时间与资源。

在排序的过程中，各元素不断接近自己的位置，如果一趟比较下来没有进行交换，就说明序列已经有序，因此要在排序过程各种设置一个标志flag判断元素是否进行过交换，从而较少不必要的比较（该优化可以在冒泡操作之后进行操作）。

小结：

（1）一共进行了（bubbleArr.length-1）次循环 ；bubbleArr.length表示冒泡数组的大小

（2）每一趟排序都是相邻元素的比较交换，且随着较大值的确定，次数在逐渐减少

（3）【优化】如果在某次排序中没有再发生一次交换，可以提前结束冒泡排序

简单选择排序

简单选择排序属于内部排序，是从待排序的数据中，按指定的规则选出来一个元素，在依规定交换位置后达到排序的目的。简单选择排序的时间复杂度是O(n2)

小结：

（1）一共进行了（selectArr.length-1）次循环 ；selectArr.length表示简单选择数组的大小

（2）每一趟排序都是一个循环，循环规则

方法A与方法B虽然都可用，但是从运算效率来看，方法B远远优于方法A。

原因在于方法A将当前最小值与之后的元素进行比较，比较后直接交换，则比较、交换的操作次数一样多

而方法B是将当前最小值先存入min变量中，让min与之后的元素进行比较，比较结束后将实际最小值存入min，实际最小值的坐标存入minIndex，比较结束后再进行交换，交换次数仅有一次

当数组大小较大时，方法B的运算速度要远远高于方法A，这也是为什么 冒泡排序和简单选择排序（方法B）的时间复杂度都是O(n2)，运算时间却相差较大。

直接插入排序

插入排序属于内部排序，是对于待排序的元素以插入的方式找寻该元素的适当位置，以达到排序的目的。基本思想是，把n个待排序的元素看成一个有序表和一个无序表，开始时有序表只包含一个元素，无序表中包含（n-1）个元素，排序过程中每次从无序表中取出一个元素，依次与有序表中元素进行比较，将它插入有序表中的合适位置，使之成为新的有序表。直接插入排序的时间复杂度是O(n2)

希尔排序

希尔排序也是一种插入排序，它是直接插入排序经过改进之后的一个 更加高效的版本，也称为 缩小增量排序。希尔排序是把记录按下标的一定增量分组，对每组直接插入排序算法排序；随着增量逐渐减少，每组包含的关键字越来越多，当增量减至1时，整个文件恰好被分成一组，算法终止。希尔排序的时间复杂度是O(n2)
