树结构基础部分

一、存储方式分析

  • 数组存储方式的分析（增删效率低）

    • 优点：通过下标方式访问元素，速度快。对于有序数组，还可以使用二分查找提高检索速度；无须为表示表中元素之间的逻辑关系增加额外的存储空间

    • 缺点：如果要检索具体某个值，或者插入值（按一定顺序）会整体移动，效率降低；当数组长度变化较大时，难以确定存储空间的容量；造成存储空间的“碎片”

    • 链式存储方式的分析（改查效率低）

      • 优点：在一定程度上对数组存储方式有优化（比如：插入一个数值结点，只需要将插入结点链接到链表中即可，不需要整体移动，同理删除效率也很高）
      • 缺点：在进行检索时，效率仍然较低（比如：检索某个值，需要从头节点开始遍历）

  • 树存储方式的分析

    能提高数据 存储、读取的效率，比如利用二叉排序树，既可以保证数据的检索速度，同时也可以保证数据的插入、删除、修改的速度

二、树的基础介绍

树的逻辑结构图

二叉树：每个结点最多只能有两个子结点的树称为二叉树，二叉树的子结点分为左结点和右结点

满二叉树：如果该二叉树的所有叶子结点都在最后一层，并且 结点总数=2n-1，n为层数，则称为满二叉树

完全二叉树：如果该二叉树的所有叶子结点都在最后一层或者倒数第二层，而且最后一层的叶子结点在左边连续，倒数第二层的叶子结点在右边连续，则称为完全二叉树

三、二叉树遍历说明

前序遍历（父左右）：先输出父结点，再遍历左子树和右子树

中序遍历（左父右）：先遍历左子树，再输出父结点，再遍历右子树

后序遍历（左右父）：先遍历左子树，再遍历右子树，最后输出父结点

具体思路说明：

  • 首先创建一棵二叉树

  • 前序遍历

    1. 先输出当前结点
    2. 如果左子结点不为空，则递归继续前序遍历
    3. 如果右子结点不为空，则递归继续前序遍历

    • 中序遍历

      1. 如果当前结点的左子结点不为空，则递归中序遍历
      2. 输出当前结点
      3. 如果当前结点的右子结点不为空，则递归中序遍历

    • 后序遍历

      1. 如果当前结点的左子结点不为空，则递归后序遍历

      2. 如果当前结点的右子结点不为空，则递归后序遍历

      3. 输出当前结点

四、二叉树查找

要求：

  1. 编写前序查找、中序查找、后序查找的方法
  2. 分别使用三种查找方法，查找特定结点
  3. 分析各种查找方式，分别比较了多少次

思路：

  • 前序查找

    1. 先判断当前结点是否为查找结点，如果是，则返回此结点
    2. 如果不是，则判断左子结点是否为空，如果不为空，则继续递归前序查找；如果左递归前序查找，找到结点，则返回该结点
    3. 如果左递归前序查找并无找到，则继续判断当前结点的右结点是否为空，如果不为空，则继续向右递归前序查找，直到返回查找结点
    4. 如果仍未找到，则返回null，此二叉树并无查找结点

    • 中序查找（以此类推）

    • 后序查找（以此类推）

五、二叉树删除

要求：

  1. 如果删除的结点是叶子结点，则直接删除该结点

  2. 如果删除的是非叶子结点，则删除该子树

思路：

由于此处的二叉树是单向的，所以进行删除操作时，应当是判断当前结点的子结点是否要删除而非当前结点是否要删除

  1. 首先考虑如果树是空树，root=null；或者树只有root一个结点，则等价将二叉树树置空

  2. 如果当前结点的左子结点不为空，并且左子结点就是删除结点，则this.left=null;并且立即返回，结束操作

  3. 如果当前结点的右子结点不为空，并且右子结点就是删除结点，，则this.right=null;并且立即返回，结束操作

  4. 如果第2，第3步仍未删除结点，那么就需要向左子树进行递归删除

  5. 如果左递归删除没有删除结点，那么就向右子树进行递归删除

六、顺序存储二叉树

从数据存储来看，数组存储方式和树存储方式可以互相转换。

所谓的顺序存储二叉树是指，存储的物理结构为数组结构，但是仍可进行前序、中序、后序遍历。

顺序存储二叉树的特点（如下图所示）：

  1. 通常只考虑完全二叉树
  2. 第n个元素的左子结点为 第(2n+1)个元素
  3. 第n个元素的右子结点为 第(2n+2)个元素
  4. 第n个元素的父结点 第 (n-1)/2 个元素

n：表示二叉树中第几个元素（按0开始编号）

七、线索二叉树

n个结点的二叉链表中含有 n+1个空指针域 2n-(n-1)=n+1，利用二叉链表中的空指针域，存放指向该结点在某种遍历次序下的前驱和后继结点的指针（这种附加的指针称为“线索”）

这种加上了线索的二叉链表称为 线索链表，相应的二叉树称为线索二叉树。根据线索性质的不同，线索二叉树可分为 前序线索二叉树、中序线索二叉树 和 后序线索二叉树三种

其实线索二叉树，就是根据遍历顺序将一颗二叉树变成一个双向链表，这样充分利用了空指针，对插入删除结点、查找结点带来方便。所以对二叉树以某种次序遍历使其变为线索二叉树的过程称为 线索化。

一个结点的前一个结点，称为前驱结点；一个结点的后一个结点，称为后继结点

规定除了结点本身就带有的 left 和 right 左右指针，增加 ltag 和 rtag两个标志域

当 ltag 为0时，left指向该结点的左子结点，为1时指向该结点的前驱；

当 rtag 为0时，right指向该结点的右子结点，为1时指向该结点的后继

说明：当线索化二叉树后，Node结点的属性left和right，有如下情况：

  • left指向的是左子树，也可能是指向前驱结点

  • right指向的是右子树，也可能是指向后继结点

八、遍历线索二叉树

线索化后，各个结点指向有变化，因此原来的遍历方式不能使用，此时需要使用新的方式遍历线索化后的二叉树，各个结点可以通过线型方式遍历，因此无需使用递归方式，这样也提高了遍历效率，遍历的次序也应当与线索化次序保持一致。
