数据结构与算法第三讲 队列（Java）

队列

一、基本定义

队列是只允许在一段进行插入操作，而在另一端进行删除操作的线性表。遵循 先进先出，后进后出。可以是一个有序列表，也可以用数组或者链表表示。

二、相关要素

  • 最大容量 maxSize

  • 队头 front 指向队头元素（）

  • 队尾 rear 指向队尾的下一个位置

    当 front=rear 时 队列空，当 rear=maxSize 时 队列满。使用一维数组模拟队列存在空间占用问题，即元素取出队列后，该元素仍然占据数组空间，造成“假溢出”。
三、循环队列

为解决普通队列的“假溢出”，引入 循环队列——头尾相接的顺序存储队列，并对普通队列作出如下调整：

  • 保留一个元素空间作为队列满的约定

  • 当队列满时，front=(rear+1)%maxSize; 当队列空时，front=rear;

  • 队列有效元素为 （rear+maxSize-front）%maxSize

数据结构第一讲 数据结构与算法绪论

Posted on 2020-03-26

数据结构与算法第一讲 数据结构与算法绪论

一、基本概念

  1. 程序设计 = 数据结构 + 算法

  2. 数据结构:是相关之间存在一种或者多种特定关系的数据元素的集合

  3. 数据元素：是组成数据的，有一定意义的基本单位，在计算机中通常作为整体处理，也称为「记录」。其中一个数据元素可以由若干个数据项组成。数据项是数据不可分割的最小单位。

  4. 数据对象：是性质相同的数据元素的集合，是数据的子集。

  5. 按照视点不同，数据结构可以分为 逻辑结构 和 物理结构，数据元素的存储关系并不能反映逻辑关系，因此需要用一个指针存放数据元素地址

  • 逻辑结构：是指数据对象中数据元素之间的相互关系

    • 集合结构：同属同一集合而已
    • 线性结构：一对一的关系
    • 树形结构：存在一对多的关系
    • 图形结构：多对多的关系

  • 物理结构（存储结构）：是指数据的逻辑结构在计算机中的存储形式

    • 顺序存储：类似于排队占位（连续）
    • 链式存储结构：把数据元素存放在任意存储单元（可连续可不连续）

    逻辑结构是面向问题的；而物理结构是面向计算机的，其基本目标就是将数据及其逻辑关系存储至计算机内存中。

  1. 数据类型：是指一组性质相同值的集合及定义在此集合上一些操作的总称。在C语言中，按照取值不同，可以分为原子类型：不可分解的基本类型和结构类型：由若干类型组合而成，可再分解
  2. 抽象数据类型(Abstract Data Type ADT)
    指一个数学模型以及定义在该模型上的一组操作（与逻辑特性有关，与计算机内部表示、实现无关，操作视具体需求而定）

二、算法

  1. 算法：算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示一个或者多个操作。算法具有五个基本特性：输入、输出、有穷性、确定性和可行性

  2. 算法设计要求 算法设计不是唯一的，但是必须符合以下要求：

    • 正确性：算法至少应该具有输入、输出和加工处理无歧义性、能正确反映问题的需求、能够得到问题的正确答案
    • 可读性：便于阅读、理解和交流
    • 健壮性：当输入数据不合法时，算法也能做出相关处理，而不是产生异常或莫名奇妙的结果
    • 时间效率高和存储量低

  3. 算法效率的度量方法

    • 事后统计方法：通过设计好的测试程序和数据，利用计算机计时器对不同算法编制的程序运行时间进行比较，从而确定算法效率高低
    • 事前分析估算方法：在计算机程序编制前，依据统计方法对算法进行估算。测定运行时间最可靠的方法就是计算对运行时间有消耗的基本操作的执行次数。运行时间与执行次数成正比

    一个用高级程序语言编写的程序在计算机上运行时所消耗的时间取决于下列因素：
    1) 算法采用的策略、方法 2) 编译产生的代码质量（由软件支持）
    3) 问题的输入规模 4) 机器执行指令的速度（与硬件性能有关）

  4. 算法时间复杂度
    在进行算法分析时，算法时间复杂度T(n)是关于问题规模n的函数，进而分析T(n)随n的变化情况并确定T(n)的数量级，记作T(n)=O(f(n))。其中f(n)是问题规模n的某个函数，判断算法效率时，函数中的常数项和其他次要项可以忽略，应该关注主项的次数。

1.首先判断是否安装 node、npm、git等必要工具，node -v,npm -v,git --version在终端输入，并获得管理员权限sudo su（在Linux系统下）

2.切换npm的国内镜像源，如淘宝镜像

3.使用cnpm安装hexo框架

4.在自定义位置创建一个空文件夹 blog:mkdir blog，并在该文件夹的路径下完成hexo的初始化:hexo init

3.分别进行清理缓存文件，生成静态文件，就可以在http://localhost:4000 进行本地访问
